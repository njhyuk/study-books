# 설명하는 변수

1. 어렵고 큰 복잡한 코드의 표현식을 이해했다면, 표현식을 추출한 후
2. 표현식의 의도가 드러나도록 변수 이름을 만들어 할당하라

```
// AS-IS
return new Point(
    ...긴 표현식...,
    ..다른 긴 표현식...
)

// TO-BE
x := ...긴 표현식
y := ...긴 표현식

return new Point(x, y)
```

* 표현식과 설명하는 변수가 분리되었기 때문에, 코드를 변경할 때 둘중 하나만 보면 된다. 
* 코드 정리에 대한 커밋과 동작 변경에 대한 커밋은 분리하라.
